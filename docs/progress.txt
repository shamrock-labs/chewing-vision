      Fix: TasksApiFaceDetector and TasksApiHandDetector classes implement
      orofac's FaceDetector / HandDetector ABCs using the modern Tasks API
      (FaceLandmarker.detect / HandLandmarker.detect in IMAGE mode). Both
      adapters are injected via ChewAnnotator's face_detector= / hand_detector=

      detected hand. ChewAnnotator.detect_hand only checks `len(results) > 0`
      (presence test), so empty inner landmarks save marshalling 21 hand
      landmarks orofac discards anyway.

      passes no timestamp. Synchronous detect() per frame.
    - Model cache shared with OursEngine at ~/.cache/chewing-vision/. Inline
      `_ensure(url, path)` helper rather than promoting to a shared module —
      that's a US-004/005/014 scope concern, not US-003's.
    - n_chews = sum(len(cycle['peaks'])). left+right+middle = sum of cycle
      direction counts. AC3 invariant (sum ≤ n_chews) is structurally
      guaranteed by orofac's own code: directions only append when a peak
      has a following valley AND vertical_motion < 0 (chew_annotator.py:225-233).
    - t_sec approximated as (cycle.start_frame + local_peak_index) / fps.
      Recorded as a Result.warnings entry because orofac's jaw_movements is
      sparse (skips hand-in-frame and face-missing frames) — confounder clips
      may drift; US-010 event-F1 must not silently trust these timestamps.
    - frames/windows/bouts = [] — orofac doesn't expose per-frame signal data
      through its public API. OursEngine covers that surface.
