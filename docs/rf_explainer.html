02 · 슬라이딩 윈도우

2초씩 잘라서 0.5초마다 슬라이딩

연속된 IMU 신호를 2초 단위로 잘라 피처를 계산하고, 0.5초씩 이동하며 반복한다.

파란 구간 = 현재 2초 윈도우  ·  점선 = 0.5초 경계
윈도우 크기: 2초 (100샘플)
씹기 한 번이 약 0.5~1초이므로, 2초 안에 2~4번의 씹기가 들어옴. 패턴을 보기 충분한 길이.
스트라이드: 0.5초 (25샘플)
윈도우가 0.5초마다 한 번씩 예측을 냄. 앱이 0.5초 단위로 "씹는 중인가?" 를 판단.
경계 오염 제거
씹기↔쉬기 전환점 앞뒤 1초 이내 윈도우는 학습에서 제외. 라벨이 섞인 구간은 AI를 혼란시킴.
03 · 피처 추출

100개 샘플 → 15개 숫자

2초 구간에서 통계값을 계산해 AI가 읽기 좋은 고정 크기 벡터로 변환한다.

rotation x
rotation_x_rms · rotation_x_std
좌우 회전 강도(RMS)와 들쭉날쭉함(표준편차)
rotation y
rotation_y_rms · rotation_y_std
상하 끄덕임 강도와 들쭉날쭉함 ★ 3·7위
rotation z
rotation_z_rms · rotation_z_std
앞뒤 기울기 강도와 들쭉날쭉함
accel x
user_accel_x_rms · user_accel_x_std
좌우 가속도 강도와 들쭉날쭉함
accel y
user_accel_y_rms · user_accel_y_std
앞뒤 가속도 강도와 들쭉날쭉함
accel z
user_accel_z_rms · user_accel_z_std
상하 가속도 강도와 들쭉날쭉함 ★ 1·2위
attitude pitch
attitude_pitch_delta_rms NEW
끄덕임 각도의 샘플 간 잔떨림. 각도 불연속 보정 포함.
attitude yaw
attitude_yaw_delta_rms NEW
좌우 돌림 잔떨림. yaw는 ±π 불연속 → atan2 보정 필수.
attitude roll
attitude_roll_delta_rms NEW
기울기 잔떨림. 피처 중요도 전체 5위 (0.0947).
# utils.py — 실제 코드 def extract_features(window): features = [] for axis in IMU_AXES: # rotation x/y/z, accel x/y/z → 12개 vals = window[axis].values features.append(np.sqrt(np.mean(vals**2))) # RMS features.append(np.std(vals)) # 표준편차 for axis in ATTITUDE_AXES: # pitch, yaw, roll → 3개 raw = np.diff(window[axis].values) diffs = np.arctan2(np.sin(raw), np.cos(raw)) # ±π 래핑 보정 features.append(np.sqrt(np.mean(diffs**2))) # delta RMS return features # 총 15개

06 · 피처 중요도

어떤 신호가 판단에 가장 중요한가?

200그루 나무가 얼마나 자주, 얼마나 효과적으로 각 피처를 사용했는지 집계. 15개 전체 표시.

accel_z가 1·2위인 이유
턱이 씹을 때 위아래(z축)로 움직임. 귓속 에어팟도 같이 상하 진동 → 가장 직접적인 씹기 신호.
attitude_roll이 5위인 이유
씹는 리듬이 머리 기울기 각도에 잔진동으로 나타남. 자이로+가속도 융합 신호라 노이즈가 적음.

coremltools 변환이란?
sklearn의 RandomForest는 Python 객체로만 존재한다. coremltools는 나무 200그루의 분기 조건(피처 이름, 기준값, 좌/우 자식)을 Apple의 protobuf 포맷으로 직렬화해 .mlmodel 파일로 저장한다. Python 런타임 없이도 동일한 판단이 가능해진다.
Xcode가 자동으로 하는 것
.mlmodel 파일을 프로젝트에 넣으면 Xcode가 Swift 래퍼 코드를 자동 생성한다. ChewingClassifierInput에 15개 피처값을 넣으면 chewing_label과 확률이 반환된다. 개발자가 ML 추론 코드를 직접 짤 필요가 없다.
온디바이스의 의미
서버 없이 아이폰 안에서만 판단이 완결된다. IMU 데이터가 외부로 나가지 않아 개인정보 보호에 유리하고, 네트워크 없이도 동작한다. CoreML은 Apple Silicon의 Neural Engine을 활용해 배터리 소모 없이 빠르게 실행된다.
# coreml_convert.py — 변환 핵심 coreml = ct.converters.sklearn.convert(clf, FEATURE_NAMES, "chewing_label") coreml.save("ChewingClassifier.mlmodel") # → 이 파일 하나에 나무 200그루의 모든 분기 기준이 담김 # → Xcode가 읽어서 Swift 코드 자동 생성 → 아이폰에서 바로 실행

중기 — 모델 품질 개선 (정확도·일반화)
👥
다수 사용자 데이터 수집
현재 7세션은 1~2인 데이터로 추정. LOSO F1이 0.49~0.97로 세션마다 편차가 크다는 게 일반화 부족의 신호. 최소 10명 이상 다양한 체형·식습관 데이터가 필요.
🧠
시계열 모델 실험
Random Forest는 각 2초 윈도우를 독립적으로 판단. LSTM·1D-CNN은 이전 씹기 패턴을 기억해 연속성을 포착 가능. 단, CoreML 변환이 복잡해지고 모델 크기가 커짐.
🍱
음식 종류 분류 (7-class)
현재 씹기/쉬기 이분법 → 딱딱함·부드러움·액체 등 세분화. SPEC US-015에 명시됨. 씹기 속도와 진동 강도 패턴이 음식마다 달라 추가 피처 설계 필요.
🎯
개인 캘리브레이션
사람마다 씹는 힘·주파수·패턴이 다름. 첫 식사 세션에서 수집한 데이터로 온디바이스 fine-tuning → 개인화된 모델로 정확도 향상 기대.
